# AU-12 — Audit Record Generation

## Control Statement

Provide audit record generation capability for the event types the system is capable
of auditing; allow designated personnel to select the event types that are to be logged;
and generate audit records for the defined events.

## Sidereal Implementation

All Sidereal components generate audit records for their respective event types.
The complete audit event inventory is defined at design time and enforced at runtime.

### Controller Manager Audit Events

| Event | Mechanism | Destination |
|---|---|---|
| Controller startup | Structured log + Kubernetes audit log | SIEM |
| Controller shutdown | Structured log + Kubernetes audit log | SIEM |
| Probe Job created | Kubernetes audit log (Job CREATE) | SIEM |
| Probe Job completed | SiderealProbeResult CREATE | SIEM |
| HMAC verification failure | SiderealProbeResult (TamperedResult) | SIEM |
| SIEM export failure | Prometheus metric + retry log | Prometheus/SIEM |
| SiderealSystemAlert created | SiderealSystemAlert CREATE | SIEM |
| SiderealSystemAlert acknowledged | Kubernetes audit log (PATCH) | SIEM |
| SiderealIncident created | SiderealIncident CREATE + IR webhook | SIEM + IR system |

### Configuration Audit Events

| Event | Mechanism | Destination |
|---|---|---|
| executionMode transition (dryRun → observe → enforce) | Kubernetes audit log (PATCH on SiderealProbe) | SIEM |
| Security-relevant Helm value changed | Helm upgrade audit record | SIEM |
| SiderealAOAuthorization created | Kubernetes audit log (CREATE) | SIEM |
| SiderealAOAuthorization expired | Controller log + SiderealSystemAlert | SIEM |

### Probe Execution Audit Events

| Event | Mechanism | Destination |
|---|---|---|
| Probe outcome: any | SiderealProbeResult CREATE | SIEM |
| Detection verification window opened | Controller log | SIEM |
| Detection backend unreachable | SiderealProbeResult (BackendUnreachable) | SIEM |
| CNI backend unreachable | SiderealProbeResult (BackendUnreachable) | SIEM |
| Admission webhook absent | SiderealProbeResult (NotApplicable) | SIEM |

### Enhancement: AU-12(1) — System-Wide and Time-Correlated Audit Trail

All audit records from all Sidereal components carry RFC 3339 timestamps synchronized
to the Kubernetes node time source. The SIEM ingest pipeline correlates records by
`probe.id` UUID, enabling end-to-end tracing of a single probe execution across all
event types above.

### Enhancement: AU-12(3) — Changes by Authorized Individuals

Designated Sidereal operators (holding `sidereal-operator` or `sidereal-live-executor`
roles) can modify probe configurations. All changes are captured in the Kubernetes
audit log with the operator's individual principal identity.

## Evidence Produced

- Complete event inventory in this control implementation
- Kubernetes audit log (all Kubernetes API interactions)
- SiderealProbeResult records (one per probe execution)
- SiderealSystemAlert records
- SiderealIncident records
- SIEM export records for all events

## Customer Responsibility

The deploying agency must:
1. Enable Kubernetes API server audit logging in their cluster (required for controller
   and configuration events above)
2. Configure Kubernetes audit log export to their SIEM alongside Sidereal's direct
   SIEM export
3. Verify that SIEM correlation by `probe.id` is configured for end-to-end tracing
